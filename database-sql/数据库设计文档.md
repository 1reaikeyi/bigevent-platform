## 数据库设计文档

| 序号 | 数据表名        | 中文名称         |
| ---- | --------------- | ---------------- |
| 1    | user            | 用户表           |
| 2    | category        | 文章分类表       |
| 3    | article         | 文章表           |
| 4    | voucher         | 优惠券主表       |
| 5    | voucher_seckill | 秒杀优惠券表     |
| 6    | voucher_order   | 优惠券订单表     |

### 1. user

user 表为用户表，用于存储博客/大事件平台的后端用户信息。用户注册后通过用户名密码登录，支持修改昵称、邮箱和头像。具体表结构如下：

| 字段名      | 数据类型      | 说明         | 备注                    |
| ----------- | ------------- | ------------ | ----------------------- |
| id          | bigint        | 主键         | 自增                    |
| username    | varchar(20)   | 用户名       | 唯一索引，非空          |
| password    | varchar(32)   | 密码         | MD5 加密存储            |
| nickname    | varchar(10)   | 昵称         | 默认空字符串            |
| email       | varchar(128)  | 邮箱         | 默认空字符串            |
| user_pic    | varchar(128)  | 头像         | 头像图片路径，默认空    |
| create_time | datetime      | 创建时间     |                         |
| update_time | datetime      | 修改时间     |                         |

### 2. category

category 表为文章分类表，用于存储用户自定义的文章分类信息。每个用户可以创建多个分类，分类名称和别名用于文章归类与展示。具体表结构如下：

| 字段名          | 数据类型     | 说明           | 备注                       |
| --------------- | ------------ | -------------- | -------------------------- |
| id              | bigint       | 主键           | 自增                       |
| category_name   | varchar(32)  | 分类名称       | 显示用，如"技术笔记"       |
| category_alias  | varchar(32)  | 分类别名       | 用于URL或查询，如"tech"    |
| create_user     | bigint       | 创建人ID       | 关联 user.id               |
| update_user     | bigint       | 更新人         | 关联 user.id               |
| create_time     | datetime     | 创建时间       |                            |
| update_time     | datetime     | 修改时间       |                            |

### 3. article

article 表为文章表，用于存储用户发布的博文/事件内容。支持草稿与发布两种状态，文章可关联分类，封面与富文本内容支持较长的存储。具体表结构如下：

| 字段名      | 数据类型       | 说明           | 备注                               |
| ----------- | -------------- | -------------- | ---------------------------------- |
| id          | bigint         | 主键           | 自增                               |
| title       | varchar(50)    | 文章标题       |                                    |
| content     | varchar(10000) | 文章内容       | 支持富文本，最长约1万字符          |
| cover_img   | varchar(128)   | 文章封面       | 封面图片路径                       |
| state       | varchar(3)     | 文章状态       | 仅允许"已发布"或"草稿"，默认"草稿" |
| category_id | bigint         | 文章分类ID     | 关联 category.id，逻辑外键         |
| create_user | bigint         | 创建人ID       | 关联 user.id                       |
| update_user | bigint         | 更新人         | 关联 user.id                       |
| create_time | datetime       | 创建时间       |                                    |
| update_time | datetime       | 修改时间       |                                    |

### 4. voucher

voucher 表为优惠券主表，用于存储商家发布的优惠券信息。支持普通代金券和秒杀券两种类型，包含价格抵扣规则和上下架状态管理。具体表结构如下：

| 字段名       | 数据类型       | 说明                 | 备注                               |
| ------------ | -------------- | -------------------- | ---------------------------------- |
| id           | bigint         | 主键                 | 自增                               |
| shop_id      | bigint         | 所属门店ID           | 可为空，表示通用券                 |
| title        | varchar(255)   | 优惠券标题           |                                    |
| sub_title    | varchar(255)   | 优惠券副标题         | 可为空                             |
| rules        | varchar(1024)  | 优惠券使用规则       | 可为空                             |
| pay_value    | bigint         | 用户支付金额         | 单位：分                           |
| actual_value | bigint         | 优惠券抵扣金额       | 单位：分                           |
| type         | tinyint        | 券类型               | 0-普通代金券，1-秒杀券             |
| status       | tinyint        | 优惠券状态           | 1-上架，2-下架，3-过期             |
| create_time  | timestamp      | 创建时间             | 默认当前时间                       |
| update_time  | timestamp      | 更新时间             | 更新时自动刷新                     |

### 5. voucher_seckill

voucher_seckill 表为秒杀优惠券表，与 voucher 表一对一关联。用于存储秒杀活动的库存和活动时间窗信息，仅在 voucher.type = 1（秒杀券）时有对应记录。具体表结构如下：

| 字段名      | 数据类型    | 说明               | 备注                           |
| ----------- | ----------- | ------------------ | ------------------------------ |
| voucher_id  | bigint      | 主键               | 关联 voucher.id，一对一        |
| stock       | int         | 秒杀库存数量       |                                |
| begin_time  | timestamp   | 秒杀活动开始时间   |                                |
| end_time    | timestamp   | 秒杀活动结束时间   |                                |
| create_time | timestamp   | 记录创建时间       | 默认当前时间                   |
| update_time | timestamp   | 更新时间           | 更新时自动刷新                 |

### 6. voucher_order

voucher_order 表为优惠券购买订单表，用于存储用户抢购/购买优惠券的订单记录。包含完整的支付链路状态（从下单到核销到退款），支持多种支付方式。具体表结构如下：

| 字段名      | 数据类型    | 说明             | 备注                                       |
| ----------- | ----------- | ---------------- | ------------------------------------------ |
| id          | bigint      | 主键             | 订单唯一ID                                 |
| user_id     | bigint      | 下单用户ID       | 关联 user.id                               |
| voucher_id  | bigint      | 购买的优惠券ID   | 关联 voucher.id                            |
| pay_type    | tinyint     | 支付方式         | 1-余额，2-支付宝，3-微信                   |
| status      | tinyint     | 订单状态         | 1-未支付，2-已支付，3-已核销，4-已取消，5-退款中，6-已退款 |
| create_time | timestamp   | 下单时间         | 默认当前时间                               |
| pay_time    | timestamp   | 支付完成时间     | 可为空                                     |
| use_time    | timestamp   | 门店核销使用时间 | 可为空                                     |
| refund_time | timestamp   | 退款完成时间     | 可为空                                     |
| update_time | timestamp   | 订单更新时间     | 更新时自动刷新                             |
